# auto-agents

Local CI powered by Claude Code agents. Runs security audits, testing, and documentation checks automatically on every commit and push.

## What it does

Three independent agents run in the background via git hooks:

| Module | Trigger | What it does |
|--------|---------|-------------|
| **auto-doc** | post-commit | Detects drift between code and docs, auto-fixes minor issues |
| **auto-test** | pre-push | Runs build + stack-specific validations (i18n, links, tests) |
| **auto-sec** | pre-push | Security audit: npm audit + code analysis by Claude |

Each module runs independently with its own lock file, log, and report.

## Supported stacks

| Stack | Build | Testing | Security |
|-------|-------|---------|----------|
| Stack | Build | Testing | Security | Docs |
|-------|-------|---------|----------|------|
| **astro** | `npm run build` | i18n consistency, internal links, frontmatter | npm audit, inline scripts, meta tags | drift detection via doc-map.md |
| **next** | `npm run build` | vitest + playwright | npm audit, auth, CSP, API endpoints, injections | drift detection via doc-map.md |

## Install

```bash
git clone https://github.com/blogos47/auto-agents.git ~/p/auto-agents
cd ~/p/auto-agents && chmod +x core.sh functions.sh install.sh modules/*.sh hooks/*.hook

# Install in your project
./install.sh ~/p/your-project
```

The installer:
1. Creates symlinks for git hooks (`post-commit`, `pre-push`)
2. Creates `logs/ci/` directory
3. Updates `.gitignore`
4. Creates `.env` with defaults

## Configuration

Each project has a `.env` file:

```bash
PROJECT_NAME="my-project"
STACK="astro"          # astro | next
MODEL="opus"           # claude model
TIMEOUT=600            # seconds
```

Optional per-module overrides: `.env.test`, `.env.sec`, `.env.doc`

## Structure

```
auto-agents/
  core.sh          — orchestrator (loads config + functions, runs module)
  functions.sh     — pure function library (lock, log, notify, git, agent)
  install.sh       — installs hooks in a project
  modules/
    auto-doc.sh    — documentation drift detection
    auto-test.sh   — build + validation
    auto-sec.sh    — security audit
  hooks/
    post-commit.hook
    pre-push.hook
```

## Manual run

```bash
# Run a specific module
~/p/auto-agents/core.sh auto-doc ~/p/your-project
~/p/auto-agents/core.sh auto-test ~/p/your-project
~/p/auto-agents/core.sh auto-sec ~/p/your-project
```

## Reports

| File | Generated by | Gitignored |
|------|-------------|------------|
| `docs/DOC_REPORT.md` | auto-doc | No (tracked) |
| `TEST_REPORT.md` | auto-test | Yes |
| `SECURITY_REPORT.md` | auto-sec | Yes |

## Requirements

- [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) (`claude` in PATH)
- bash 4+
- git
- python3 (for stats parsing)
- Node.js (for npm projects)
